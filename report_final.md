## Part I: Music Genre Classification

### 1. Introduction

This coursework studies music genre classification on the GTZAN dataset. The task is a ten-class classification problem containing genres such as blues, classical, country, disco, hiphop, jazz, metal, pop, reggae, and rock. Six neural network classifiers were implemented and compared: a fully connected baseline, two convolutional models, a convolutional model trained with RMSProp, an LSTM audio classifier, and an LSTM classifier trained with GAN-based data augmentation.

The aim of the experiments is not only to obtain high accuracy, but also to understand how different design choices affect performance. In particular, the experiments compare fully connected layers against convolutional feature learning, the effect of batch normalisation, the effect of changing the optimiser, the use of recurrent models for audio features, and the usefulness of synthetic samples generated by a GAN.

### 2. Code Structure and Dataset

The coursework experiments are managed through the Jupyter notebook `coursework1_runner.ipynb`. To keep the notebook readable, the model definitions, dataset classes, training loop, evaluation functions, GAN augmentation, and curve plotting are implemented in `music_genre_coursework.py`. The notebook imports and runs the six required architectures in sequence: `Net1`, `Net2`, `Net3`, `Net4`, `Net5`, and `Net6`.

For `Net1` to `Net4`, the spectrogram images in `images_original` were used. Each image was resized to `180 x 180` when loaded, converted to a tensor, and normalised. The dataset was split into training, validation, and test sets using a `70% / 20% / 10%` ratio.

For `Net5` and `Net6`, audio-derived features from `features_3_sec.csv` were used. Each 30-second track is represented as a sequence of ten 3-second feature vectors. This gives the LSTM a short temporal sequence while avoiding the instability of training directly from raw waveforms on a small dataset. Features were standardised using statistics from the training set only.

### 3. Model Architectures

`Net1` is a fully connected network with two hidden layers. The input image is flattened before passing through dense layers with ReLU activations and dropout. This provides a simple baseline, but it does not explicitly use the local structure of spectrogram images.

`Net2` is a convolutional neural network based on the architecture suggested in the coursework brief. It uses four convolutional layers with max pooling after pairs of convolutional layers. This allows the model to learn local time-frequency patterns in the spectrogram.

`Net3` uses the same CNN structure as `Net2`, but adds batch normalisation after convolutional layers. Batch normalisation is expected to make training more stable and improve convergence.

`Net4` uses the same architecture as `Net3`, but replaces Adam with RMSProp. This isolates the effect of the optimiser while keeping the network architecture fixed.

`Net5` is a two-layer bidirectional LSTM trained on the 3-second audio feature sequences. The final forward and backward hidden states are concatenated and passed to a small classifier. Layer normalisation and dropout are used to reduce overfitting.

`Net6` uses the same LSTM classifier as `Net5`, but augments the training set using a conditional GAN. The GAN is trained on flattened audio feature sequences conditioned on genre labels. The generated samples are reshaped back to the original sequence format and added to the real training examples.

### 4. Training Setup

All models were implemented in PyTorch and trained using cross-entropy loss. `Net1`, `Net2`, and `Net3` used Adam with a learning rate of `1e-4`. `Net4` used RMSProp with the same learning rate. For the image models, training was performed for both `50` and `100` epochs as required by the coursework brief.

For `Net5` and `Net6`, Adam with a learning rate of `1e-3` was used. Both audio models were trained for `80` epochs. During training, both loss and accuracy were recorded for the training and validation sets. The checkpoint with the best validation accuracy was used for final testing. Accuracy, error rate, and loss curves were saved for each model.

Its per-class report showed strong performance on metal, pop, blues, classical, and jazz, while rock was the weakest class. The best audio-based model was `Net5`, with a test accuracy of `0.720`. It performed well on jazz, metal, classical, and blues, but was weaker on hiphop, reggae, and rock.

### 6. Discussion

The fully connected baseline was the weakest model. This is expected because flattening the spectrogram removes the two-dimensional local structure of the image. Music spectrograms contain local visual patterns that correspond to frequency bands, rhythmic textures, and timbral structures. A fully connected network has to learn these relationships from a very large vector, which makes the task more difficult.

`Net2` improved clearly over `Net1`, showing that convolutional feature extraction is much more suitable for spectrogram images. The test accuracy increased from `0.366` for the best `Net1` run to around `0.545` for `Net2`. This supports the idea that CNNs can learn useful local time-frequency patterns.

Adding batch normalisation in `Net3` gave the largest improvement among the image models. `Net3` reached `0.752` test accuracy after 100 epochs, which was the best overall result. The accuracy curves also showed more stable learning than the plain CNN. This suggests that batch normalisation helped optimisation and made the convolutional model generalise better.

`Net4` used the same architecture as `Net3` but changed the optimiser to RMSProp. Its performance was strong, especially at 50 epochs, but it did not outperform `Net3` at 100 epochs. This suggests that the architecture and normalisation were more important than the optimiser choice in this experiment. RMSProp was competitive, but Adam gave the best image result.

The comparison between 50 and 100 epochs shows that longer training can help, but not always. `Net1`, `Net3`, and `Net4` improved from 50 to 100 epochs, while `Net2` slightly decreased. This indicates that more epochs can improve performance when the model is still learning useful features, but it can also lead to overfitting or unstable validation behaviour.

`Net5` achieved strong performance using audio feature sequences, with `0.720` test accuracy. This shows that temporal audio features are useful for genre classification. The result is close to the best image-based models, suggesting that both spectrogram images and audio feature sequences contain useful information for this task.

`Net6` did not improve over `Net5`. Although GAN augmentation is intended to increase training diversity, the generated feature sequences may not have been realistic or label-consistent enough. This is a useful result because it shows that data augmentation quality matters. Adding synthetic examples does not automatically improve performance, especially when the generator is simple and the dataset is small.

Overall, the results show that models which better match the structure of the input perform better. CNNs are better than fully connected networks for spectrogram images, batch normalisation improves CNN training, and LSTMs can model temporal audio features effectively. GAN augmentation is promising in theory, but in this implementation it did not outperform the non-augmented LSTM.

### 7. Conclusion

This coursework implemented and compared six neural architectures for GTZAN music genre classification. The best result was obtained by `Net3`, a CNN with batch normalisation trained for 100 epochs. The LSTM model `Net5` also performed well, showing that audio feature sequences are effective for genre classification. The experiments demonstrate the importance of choosing architectures that match the input representation. They also show that longer training and data augmentation need to be interpreted carefully, because both can lead to overfitting or unstable validation performance.

## Part II: Reflection on Deep Learning Technologies

### Topic: Representation Learning in Deep Learning

Representation learning is important because it allows a model to learn useful features automatically instead of relying only on manually designed features. This is especially valuable for complex data such as images, audio, and text. In this coursework, the difference between `Net1` and the CNN models demonstrates this clearly. The fully connected model receives the same image information, but it does not use the spatial structure of the spectrogram. The CNNs learn more suitable representations and achieve much better performance.

Several current deep learning technologies are related to representation learning. CNNs are effective for image-like data because they learn local filters and hierarchical features. RNNs and LSTMs are useful for sequential data because they can model temporal dependencies. Transformers and attention mechanisms are now widely used for sequence modelling because they can capture long-range relationships more flexibly than traditional recurrent networks. Generative models such as GANs and diffusion models also learn representations that can be used for synthesis, augmentation, and unsupervised learning.

At my current stage, I am confident implementing basic feedforward networks, CNNs, dropout, batch normalisation, common optimisers such as Adam and RMSProp, and LSTM-based classifiers in PyTorch. I can also implement a simple GAN, as used in this coursework, but I find GANs more difficult to train reliably. The main difficulties are training instability, mode collapse, and balancing the generator and discriminator. I understand the basic idea of transformers, but implementing them efficiently and tuning them well is still more challenging because attention mechanisms, positional encoding, and larger-scale training require more careful design.

The positive impact of representation learning is that it has greatly improved performance in areas such as computer vision, speech recognition, medical diagnosis, recommendation systems, and scientific data analysis. It reduces the need for hand-crafted features and can discover patterns that are difficult for humans to design manually. However, there are also negative impacts. Deep learning models can be computationally expensive, difficult to interpret, and highly dependent on the quality and fairness of the training data. If the training data is biased, the learned representation can also be biased. Generative models can additionally be misused to create misleading synthetic media.

In the future, I expect representation learning to remain central to deep learning, but with more emphasis on efficiency, interpretability, and responsible use. Better representations should require less labelled data, generalise more reliably, and be easier to understand. For students and practitioners, this means it is important not only to use high-level libraries, but also to understand why different architectures are suitable for different types of data.
